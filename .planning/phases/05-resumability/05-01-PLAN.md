<objective>
Rendre les runs de l'agent resumables via un checkpointer SqliteSaver (TOOL-07).

`build_graph()` accepte un `checkpointer` optionnel et le passe à `.compile()`. `run()` accepte un `thread_id` optionnel : si fourni, il construit un SqliteSaver sur `CHECKPOINT_DB`, compile le graphe avec, et invoque avec `config={"configurable": {"thread_id": thread_id}}` — le checkpointer persiste plan/findings/iterations/current_step dans SQLite. Relancer le même `thread_id` reprend l'état au lieu de replanifier.

Point critique (D-05) : la connexion DuckDB est `UntrackedValue` → jamais checkpointée. À CHAQUE run avec `thread_id`, `run()` ré-injecte une connexion DuckDB fraîche dans l'état initial (le checkpointer restaure le reste de l'état logique, la ressource I/O est recréée).

Purpose: Un run interrompu reprend là où il s'est arrêté.
Output: config.py (CHECKPOINT_DB), graph.py (checkpointer + thread_id + ré-injection conn), tests/test_resumability.py.
</objective>
