<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP /ask body → run() | Input utilisateur non fiable (longueur, contenu) |
| question → LLM → SQL → DuckDB | SQL généré exécuté sur la base (prompt injection write/DDL) |
| nom de fichier CSV → DDL interpolée | Nom de fichier interpolé dans CREATE TABLE |
| png_path → HTML attribut | Valeur injectée dans <img src> (XSS) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | _execute_subquestion (SQL write/DDL) | mitigate | _is_write_sql rejette DROP/DELETE/… avant exec, finding d'erreur, pas de retry (D-04) |
| T-07-06 | Tampering | loader DDL nom de table | mitigate | regex `^[A-Za-z_][A-Za-z0-9_]*$` avant interpolation, skip+warning sinon (D-05) |
| T-07-07 | Information Disclosure | /ask réponse findings | mitigate | sql/rows/columns retirés par défaut ; debug opt-in (D-06) |
| T-07-08 | Denial of Service | /ask question length | mitigate | max_length=2000 borne l'input (D-06) |
| T-07-09 | Elevation/XSS | render.py img src | mitigate | html.escape(png_path, quote=True) (D-07) |
</threat_model>
