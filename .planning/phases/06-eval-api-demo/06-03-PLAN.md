  <action>
    Create `src/dataagent/render.py` (D-06):
    - `import markdown` and `from dataagent.config import REPORTS`.
    - `render_html(report_md: str, findings: list[dict] | None = None) -> str`:
      1. `body = markdown.markdown(report_md or "", extensions=["extra"])` (extra → tables/etc.).
      2. Collect viz png_paths from findings (entries with key "png_path"). For each png_path NOT
         already present in `report_md` (the synthesizer may already embed it), append an
         `<img src="{png_path}" alt="graphe">` to the body so viz images are embedded (D-06).
      3. Wrap in a full standalone HTML document: `<!DOCTYPE html>` + `<head>` with a `<style>`
         block implementing the SolidScale rebrand — sober clean palette (neutral background,
         restrained accent), readable typography (system/serif stack, comfortable line-height,
         max-width container), styled tables/images. Keep CSS inline in the <head> so the file is
         standalone (no external assets besides the local PNGs).
    - `render_report_to_file(report_md: str, findings: list[dict] | None = None, name: str = "report") -> Path`:
      `REPORTS.mkdir(parents=True, exist_ok=True)`; write `render_html(...)` to `REPORTS / f"{name}.html"`
      (utf-8); return the Path.
    - Pure functions, no LLM, no network. Type-hint all signatures. One-line docstrings.
    Create `tests/test_render.py`:
    - test render_html returns a "<!DOCTYPE html>" doc containing report text and a <style> block.
    - test a png_path in findings (not in markdown) produces an `<img` with that src.
    - test render_report_to_file writes REPORTS/<name>.html (use a unique name or tmp via
      monkeypatching REPORTS), file exists, content == render_html output. Real file I/O (no mock),
      no LLM (D-07 — quota-free).
    - test empty report_md does not crash.
  </action>
